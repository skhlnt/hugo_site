之前我在LLM刚出来的时候曾经试着求助它，但是效率似乎并不高于偷窥真人社区。前两天我看到说StackOverflow已经凉透了，加上最近一段时间Copilot用得比较顺手（虽然还是不敢尝试ClawdBot），就想着改一波博客的配置。

### 外观调整

之前一直使用在网上找的 PaperMod 主题（因为相对简洁便利而且代码比较清楚好扒拉），只改了字体。新的改动包括：

- 将圆角矩形元素改为正常矩形。清除 iOS 歪风，更为贴近我的安卓审美。
- 删除夜晚模式。我相信我的眼球已经足够残废，如果我都能接受亮色背景，那你一定也可以。
- 更改配色方案。白色更白，黑色更黑，在适当的地方加入橘子相关颜色。适合高雅人士。

以前我对于hugo肚子里在干什么没啥概念。现在我意识到之前网站有一些地方很丑陋，菜单栏的一部分页面有自己的 ``/layouts/_default/{menu_page_name}.html``而一部分则没有（PaperMod主题自动生成）。这导致我的一些格式要求一开始不能很好地贯彻。

### 增加 Gallery

以前这个网站使用过 hexo，当时我比较闲，花了一些时间，在网站里面加入了一些我自己画着玩的地图，比如[这种](https://www.andrepimpo.wang/archives/3db1f82d.html/)。但是，后来我转而使用 hugo，并在 vercel 上配置网站，诸事繁冗，就没有正常地转移这些地图。GPT5-Codex帮助我重新制作了这些地图，并搭建了[menu/gallery](https://www.andrepimpo.wang/gallery/)栏目来存放它们。

可以使用公开的 ``gdal2tiles.py``把图片切成小块。在 ``/static/images/gallery/maps``中，我使用一个脚本来预处理所有的 ``.png``文件，生成缩略图、切片文件夹并更新 ``/data/gallery.yml``。

### 使用 Cloudflare

地图切片文件数目众多，超出上限，使得 Vercel 免费服务拒绝托管我的网站。因此，我不得不求助于 Cloudflare 的 CDN 服务。Cloudflare 提供 R2 存储桶，其道理和 AWS S3 类似，据红迪网所说，Cloudflare 给它起这个名字只是为了压 S3 一头。R2 提供10 GB 免费存储和每月1e7次访问，非常够用。

我接触了Rclone，使用这个命令行工具可以批量把图片文件传到 R2 桶里。

Cloudflare 并不是完全在做慈善，如果你要经常取桶里的文件看，最好还是挂载一个 Cloudflare domain。我本来的域名是托管在阿里云的，于是我需要去那边把 DNS 提供商从默认改为 Cloudflare。现在基建确实很好，据称需要花24-48h的事情，5分钟就搞定了。

### 谨慎使用LLM

大部分时候，体验还不错。在处理我完全不熟悉的项目（hugo 网站）时，我遇到的问题主要是：

- 过度依赖 context，容易陷入思维定式。Claude 4.5 Sonnet 和 Gemini 2.5 Pro 都遇到过，所以可能是普遍现象。例如，在聊天的前半部分，模型可能出于节省 token 的想法，没有读一些文件，拍脑袋想了一个有缺陷的办法。后面指出问题叫他改，他只会在错误上面叠床架屋，弄出极其臃肿、不太稳定、扩展性很差的解决方案。最后问多了问烦了之后我自己都差不多弄明白原理了，开一个新聊天说我自己的想法，问他该怎样写具体的代码，他就能给出不错的答案。
- 不太谨慎。例如，本来我的 PaperMod theme 是用 ``git submodule`` 挂载的，并且以前比较漫不经心地在本地的 ``themes/PaperMod/``里面做了一些改动（这些改动当然不可能推到 PaperMod 本身的仓库；永远不要这样做）。当 Claude 建议我使用 Github Page 做图床的时候，它提出了类似

  ```bash
  git checkout --orphan map-storage
  git rm -rf .
  ```
  这样的建议。这导致即使你后来又 checkout 回到主分支，你仍然需要重新从远程仓库加载 submodule，所以在 ``themes/PaperMod/``里面的改动也不会回来，它们永久消失了。

  一开始我其实问了这样干会不会有影响，但是他非常自信地认为不会有任何问题。总的来说，如果人类的水平比他低很多的话，他是不会考虑到你面临的一些问题的，所以可能还是要懂得足够多才行。
